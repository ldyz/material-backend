# AOA网络图更新说明

## 最新架构变更（2025年2月）

### 🔄 起始节点与虚任务架构

**变更概述**：
从原来的"起始线在X=0"的架构，重构为"项目开始节点S通过虚任务连接到起始任务共享节点"的新架构，更符合AOA规范。

**旧架构（已废弃）**：
```
     起点 |
         | (X=0)
    任务1 ●～～～～●────────> ●────
    任务2 ●～～～～～～●──────> ●────
```

**新架构（当前）**：
```
     S ○- - - - -> ●────────> ●────
   (项目开始)      (共享起始节点)
                    任务1开始节点
                    任务2开始节点
```

**核心改进**：
1. **项目开始节点**：新增节点编号0（S），位于项目开始日期（X=0）
2. **虚任务连接**：项目开始节点S通过虚任务（dummy task）连接到起始任务的共享开始节点
3. **共享起始节点**：按照R11规范，所有没有前置依赖的起始任务共享同一个开始节点
4. **Lag参数**：虚任务的lag参数表示项目开始日期到最早任务开始日期的天数

**好处**：
- 符合AOA规范，使用标准的虚任务表示依赖关系
- 起始节点就是起始线本身，无需额外的垂直线
- 通过lag参数清晰展示项目启动等待时间
- 所有连接都是任务（实任务或虚任务），更符合双代号网络图概念

**实现原理**：
1. 创建项目开始节点（节点编号0，X=0）
2. 根据R11规范，所有没有前置依赖的任务共享同一个开始节点
3. 在虚任务系统中添加从S到共享起始节点的虚任务
4. 虚任务类型为FS，lag为任务开始天数

**代码实现**：
```javascript
// 创建项目开始节点
const projectStartNode = {
  id: `node-project-start`,
  number: 0,
  x: 0,
  y: projectStartNodeY,
  isProjectStart: true,
  connectedStartNode: sharedStartNode, // 记录连接的共享起始节点
  // ...
}

// 在虚任务系统中添加从S到共享起始节点的虚任务
const projNode = eventNodes.value.find(node => node.isProjectStart)
if (projNode && projNode.connectedStartNode) {
  const sharedStartNode = projNode.connectedStartNode
  const lagDays = Math.round(sharedStartNode.baseX / props.dayWidth)

  deps.push({
    key: `dep-project-start-to-initial`,
    path: calculateOptimizedPath(...),
    isDummy: true,
    depType: 'FS',
    lag: lagDays
  })
}
```

## 已实现的功能

### ✅ 1. 项目开始节点与虚任务连接

**功能说明**：
1. **项目开始节点**：在项目开始日期（X=0）创建节点S
2. **虚任务连接**：通过虚任务将S连接到起始任务的共享开始节点
3. **Lag参数**：虚任务的lag表示项目开始到首个任务开始的天数

**视觉效果**：
```
        S ○- - - FS+5天 - -> ●────────> ●────
      (项目开始)           (共享起始节点)  任务1
                                 任务2
                                 任务3
```

**架构细节**：
- 项目开始节点S在X=0（项目日期）
- 共享起始节点在最早任务的实际开始时间X坐标
- 虚任务使用虚线表示，标记为"FS+N天"
- 如果最早任务从项目开始日期开始（lag=0），虚任务仍然显示但标记为"FS"

**优点**：
- 符合AOA规范，使用标准的虚任务表示依赖关系
- 起始节点就是起始线本身
- 通过lag参数清晰展示项目启动等待时间
- 所有连接统一为任务（实任务或虚任务）

### ✅ 2. 恢复R11规范的共享节点模式

**修改前**：每个任务都有独立的开始和结束节点
```javascript
node-45-start, node-45-end
node-46-start, node-46-end
```

**修改后**：相同紧前/紧后条件的任务共享节点
```javascript
// 如果任务B和任务C都依赖任务A，它们共享同一个开始节点
node-1 (任务A.end)
node-2 (任务B.start, 任务C.start) // 共享节点
```

**优点**：
- 符合AOA R11规范
- 节点数量更少
- 图形更简洁

### ✅ 3. 多方向引出线（上/中/下）

**实现原理**：
1. 判断目标任务相对于共享节点的Y坐标位置
2. 选择引出方向：
   - **上方** (`dy < -10`): 向上引出 → 水平移动 → 向下到目标行
   - **下方** (`dy > 10`): 向下引出 → 水平移动 → 向上到目标行
   - **水平** (`|dy| ≤ 10`): 直接水平连接

**路径示例**：
```
向上引出：节点 → 上15px → 水平 → 目标Y → 圆边缘
向下引出：节点 → 下15px → 水平 → 目标Y → 圆边缘
水平连接：节点右侧 → 圆边缘
```

### ✅ 5. 支持四种甘特图逻辑关系

**支持的关系类型**：
- **FS (Finish-to-Start)**: 前置任务结束后，后续任务才能开始（+lag天）
- **SS (Start-to-Start)**: 前置任务开始后，后续任务才能开始（+lag天）
- **FF (Finish-to-Finish)**: 前置任务结束后，后续任务才能结束（+lag天）
- **SF (Start-to-Finish)**: 前置任务开始后，后续任务才能结束（+lag天）

**实现方式**：
- 根据关系类型确定连接的节点（开始或结束）
- 虚任务使用虚线表示
- 标记关系类型和lag值

## 规则遵循

### R4: 节点编号规则
- 箭线起点编号 < 终点编号
- 自动检测违规并警告

### R11: 共享节点规则
- 相同紧前条件 → 共享开始节点
- 相同紧后条件 → 共享结束节点

### R19: 箭线方向规则
- 箭线只能向右（x2 ≥ x1）
- 自动检测违规并纠正

  date: '2025-01-01',
  tasks: {
