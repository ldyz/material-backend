# 网络图路径设计约束

## 核心约束

### 1. 线条重叠避免

**动态优先级调整**：
- 追踪每个节点的出线方向使用情况（右/上/下）
- 当计算新路径时，检查该节点的出线方向使用情况
- 对已使用的出线方向降低优先级（每个已有线条降低30优先级）
- 这样新路径会优先选择未使用或使用较少的出线方向

**实现机制**：
```javascript
// 计算每个方向的优先级惩罚
const rightPenalty = exitUsage.right * 30  // 每个已有的右侧出线降低30优先级
const topPenalty = exitUsage.top * 30
const bottomPenalty = exitUsage.bottom * 30

// 应用到路径优先级
priority: 100 - rightPenalty  // 右出线优先级 = 基础优先级 - 使用惩罚
```

**效果**：
- 当右侧已有多个线条时，上/下方向优先级会相对更高
- 系统会自动选择较少使用的出线方向
- 减少线条重叠，提高图形可读性

### 2. 正交路径约束
所有连线必须使用**正交路径**（Orthogonal Routing），即：
- ✅ 只使用**水平线段**（Horizontal segments）
- ✅ 只使用**垂直线段**（Vertical segments）
- ✅ 所有转角必须是**90度直角**
- ❌ 不允许使用斜线或对角线
- ❌ 不允许使用曲线
- ❌ 不允许使用小于或大于90度的转角

### 2. 箭头指向约束
所有箭头必须**指向圆心**：
- 路径终点计算到圆心位置（toX, toY）
- 箭头标记的 `refX` 设置为 `nodeRadius + arrowLength - offset`
  - 任务箭头（箭头长度9px）：`refX = nodeRadius + 7`
  - 依赖箭头（箭头长度8px）：`refX = nodeRadius + 6`
- 视觉效果：箭头尖端刚好接触圆边缘，视觉上指向圆心

## 实现细节

### 箭头标记配置

```html
<!-- 箭头标记 - refX根据节点半径动态计算 -->
<marker
  id="arrowhead-task"
  :refX="nodeRadius - 2"
  refY="3.5"
  orient="auto"
  markerUnits="userSpaceOnUse"
>
  <path d="M0,0 L0,7 L9,3.5 z" fill="#64B5F6" />
</marker>
```

**关键参数**：
- `refX="nodeRadius - 2"`：箭头尖端距离路径终点的偏移量
- 当路径终点在圆心时，箭头尖端会刚好停在圆边缘附近
- `-2` 的偏移确保箭头不会插入圆内

### 路径类型

#### 1. 实工作路径（任务箭头）
使用 `calculateOptimizedPath()` 函数生成：
- **终点在圆心**：`(toX, toY)`
- 水平连接：0个转角
- L形路径（垂直→水平 或 水平→垂直）：1个90度转角
- Z形路径（垂直→水平→垂直）：2个90度转角

#### 2. 虚工作路径（依赖关系）
使用 `calculateMultiDirectionPath()` 函数生成：
- **终点在圆心**：`(toX, toY)`
- 支持6种进出方向组合
- 所有路径都由水平和垂直线段组成
- 优先选择弯折最少的路径

#### 3. 临时连线（创建依赖时）
使用 `calculateOrthogonalPath()` 函数生成：
- **终点在圆心**：`(endX, endY)`
- 简单的正交路径（水平+垂直）
- 保证90度转角

### 方向选择

**起点出线方向**：
- 右侧（默认）
- 上侧
- 下侧

**终点方向**：
- **圆心**（所有路径的终点）
- 箭头标记自动处理到圆边缘的偏移

### 路径优化策略

路径选择按以下优先级：
1. **弯折最少**（最重要）
2. **右侧出线优先**（在弯折数相同时，优先从右侧出线）
3. 路径长度最短
4. 左进优先（在进线方向选择时）

### 出线方向优先级

**起点出线方向**（按优先级排序）：
1. **右侧**（优先级最高） - 从节点右侧出线
2. 上侧 - 从节点上侧出线
3. 下侧 - 从节点下侧出线

**终点方向**：
- **圆心**（所有路径的终点）
- 箭头标记自动处理到圆边缘的偏移

### 路径方案示例

```
✅ 优先方案1（0弯折，右侧出线）：
    [节点1] ─────────────────→ [节点2]圆心

✅ 优先方案2（1弯折，右侧出线）：
    [节点1]
       │
       └────────────→ [节点2]圆心

✅ 优先方案3（2弯折，右侧出线回绕）：
    [节点1]
       │
       ├────┐
       │    └────→ [节点2]圆心
       │
    绕过障碍

⚠️ 备选方案（非右侧出线，仅在必要时使用）：
    [节点1]
       │
       ├──────────→ [节点2]圆心
```

## 视觉效果

```
✅ 正确的路径（90度转角，箭头指向圆心）：
    从节点出发 → 水平移动 ─────→ → 垂直移动 ↓ → 水平移动 ─────→ [圆心]
                                                                  ↓
                                                              箭头尖端

❌ 错误的路径（锐角或钝角）：
    从节点出发 → 斜线 ↗      到达节点  ❌
    从节点出发 → 曲线 ╰─      到达节点  ❌
```

## 代码约束

所有路径计算函数必须遵守：

```javascript
// ✅ 正确 - 只使用水平和垂直线段，终点在圆心
`M ${x1} ${y1} L ${x2} ${y1} L ${x2} ${y2} L ${toX} ${toY}`
//                                           ↑______终点在圆心____↑

// ❌ 错误 - 使用斜线
`M ${x1} ${y1} L ${x2} ${y2}`  // 直接对角线

// ❌ 错误 - 使用曲线
`M ${x1} ${y1} Q ${cx} ${cy} ${x2} ${y2}`  // 二次贝塞尔曲线

// ❌ 错误 - 终点不在圆心
`M ${x1} ${y1} L ${x2} ${y1} L ${x2 - radius} ${y2}`  // 终点在圆边缘
```

## 维护说明

在修改路径计算代码时，务必确保：
1. 新增的路径方案只使用水平和垂直线段
2. 所有转角都是90度
3. **路径终点必须在圆心**（toX, toY）
4. 箭头标记的 `refX` 必须设置为 `nodeRadius + 7`（任务箭头）或 `nodeRadius + 6`（依赖箭头）
5. **右侧出线不是强制优先级**，当右侧有其他线条时应优先使用其他方向
6. 在弯折数相同时，优先级：右侧出线 > 上/下出线（但右侧不强制）
7. 测试各种节点位置组合，确保不会出现斜线
8. 验证路径符合AOA网络图规范

## 优先级设置参考

在路径方案中设置优先级时，参考以下标准：
- **100**: 0弯折，右侧出线
- **95**: 1弯折，上/下出线（略微低于右侧，但很接近）
- **90**: 1弯折，右侧出线
- **80**: 2弯折，右侧出线
- **70**: 2弯折，上/下出线

注意：右侧出线不是强制优先级，各方向优先级较平衡，以避免线条重叠。
